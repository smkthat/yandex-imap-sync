## 1. Подготовка

Убедитесь, что у вас установлены [Docker](https://docs.docker.com/get-docker/) и [Make](https://www.gnu.org/software/make/).

1. Скопируйте шаблон `.env.example` в `.env` (если создавали) или просто создайте `.env`:

   ```bash
   SOURCE_MAILBOX_TYPE=login
   SOURCE_IMAP_LOGIN=source@example.com
   SOURCE_IMAP_APP_PASSWORD=your_source_app_password_here

   TARGET_MAILBOX_TYPE=shared
   TARGET_SHARED_MAILBOX_DOMAIN=example.org
   TARGET_SHARED_MAILBOX_NAME=info
   TARGET_DELEGATE_LOGIN=john.doe
   TARGET_IMAP_APP_PASSWORD=your_target_delegate_app_password_here
   ```

   `TARGET_IMAP_APP_PASSWORD` — это пароль приложения для Яндекс Почты от аккаунта сотрудника `john.doe@example.org`, которому выдан доступ к общему ящику. Для общего ящика `info@example.org` отдельный пароль не нужен.

   Скрипт рассчитан на Yandex Mail/Yandex 360. По умолчанию используется IMAP-сервер `imap.yandex.ru`. При необходимости его можно переопределить переменной `YANDEX_IMAP_SERVER`.

## 2. Сборка образа

Для сборки Docker-образа выполните:

```bash
make build
```

## 3. Запуск

Используйте `make` для управления контейнером:

- **Запуск с логами в реальном времени** (рекомендуется):

  ```bash
  make run-logs
  ```

- **Запуск в фоновом режиме**:

  ```bash
  make run
  ```

- **Просмотр логов** (если запущен в фоне):

  ```bash
  make logs
  ```

- **Остановка контейнера**:

  ```bash
  make stop
  ```

### Переопределение параметров

Вы можете передать дополнительные аргументы `PARAMS` прямо в команду запуска, если нужно изменить режим синхронизации:

```bash
# Пример запуска проверки логина:
make run-logs PARAMS="--justlogin"

# Пример запуска только для папок:
make run-logs PARAMS="--automap --useheader Message-Id --errorsmax 5 --justfolders"
```

### Доступные режимы `PARAMS`

`PARAMS` — это аргументы, которые напрямую передаются в `imapsync`. Для типового переноса используйте один из режимов ниже.

| Режим                          | Команда                                                              | Что делает                                                                               |
| ------------------------------ | -------------------------------------------------------------------- | ---------------------------------------------------------------------------------------- |
| Проверка логина                | `--justlogin`                                                        | Проверяет подключение к source и target, письма и папки не переносит.                    |
| Dry-run папок                  | `--dry --justfolders --automap --useheader Message-Id --errorsmax 5` | Проверяет маппинг папок и показывает, какие папки будут созданы, без реальных изменений. |
| Создать/синхронизировать папки | `--justfolders --automap --useheader Message-Id --errorsmax 5`       | Создает недостающие папки на target, но письма не переносит.                             |
| Полный перенос                 | `--automap --useheader Message-Id --errorsmax 5`                     | Создает недостающие папки и переносит письма с вложениями.                               |

Рекомендуемый порядок запуска:

```bash
make run-logs PARAMS="--justlogin"
make run-logs PARAMS="--dry --justfolders --automap --useheader Message-Id --errorsmax 5"
make run-logs PARAMS="--automap --useheader Message-Id --errorsmax 5"
```

Пояснения к основным опциям:

- `--automap` сопоставляет стандартные папки вроде `Sent`, `Drafts`, `Spam`, `Trash`.
- `--useheader Message-Id` помогает не дублировать уже перенесенные письма при повторном запуске.
- `--errorsmax 5` останавливает процесс после 5 ошибок.
- `--dry` показывает план действий без изменений на target.
- `--justfolders` работает только с папками и не переносит письма.

Не добавляйте `--delete1` или `--delete2`, если не хотите удалять письма на source или target.

### Режимы ящиков Yandex

Скрипт поддерживает два режима для каждого ящика:

- `login` — обычный ящик, где IMAP-логин совпадает с email-адресом.
- `shared` — общий ящик Yandex 360, где IMAP-логин собирается через сотрудника с доступом.

Пример обычного ящика:

```env
SOURCE_MAILBOX_TYPE=login
SOURCE_IMAP_LOGIN=source@example.com
SOURCE_IMAP_APP_PASSWORD=your_source_app_password_here
```

Пример общего ящика:

```env
TARGET_MAILBOX_TYPE=shared
TARGET_SHARED_MAILBOX_DOMAIN=example.org
TARGET_SHARED_MAILBOX_NAME=info
TARGET_DELEGATE_LOGIN=john.doe
TARGET_IMAP_APP_PASSWORD=your_target_delegate_app_password_here
```

Для общего ящика скрипт собирает технический IMAP-логин в формате:

```text
домен/логин_сотрудника/имя_общего_ящика
```

Например, для `info@example.org` и сотрудника `john.doe` получится:

```text
example.org/john.doe/info
```

Поддерживаемые направления синхронизации:

- `login -> login`
- `login -> shared`
- `shared -> login`
- `shared -> shared`

*Процесс синхронизации может быть долгим. При использовании `make run-logs` контейнер автоматически удалится после завершения работы скрипта.*

## 4. Переадресация новых писем

После переноса старых писем настройте переадресацию, чтобы новые входящие письма на `source@example.com` попадали в общий ящик `info@example.org`.

Переадресация работает только для новых входящих писем. Старые входящие, отправленные и исходящие письма она не переносит — для этого используется `imapsync`.

### Рекомендуемый способ: правило в админке Yandex 360

1. Откройте [кабинет организации](https://admin.yandex.ru).
2. Перейдите в `Почта -> Правила для писем`.
3. Нажмите `Добавить правило`.
4. В блоке условий укажите:
   - параметр: `Кому`;
   - строгость: `совпадает` или `содержит`;
   - значение: `source@example.com`.
5. В блоке действий выберите `Переслать` и укажите `info@example.org`.
6. Сохраните правило, затем сохраните список правил.

### Альтернатива: правило в ящике-источнике

Если нет доступа к админке организации:

1. Войдите в почту `source@example.com`.
2. Откройте `Все настройки -> Правила обработки писем -> Создать правило`.
3. Настройте правило для новых входящих писем.
4. Включите действие `Переслать по адресу` и укажите `info@example.org`.
5. Если Яндекс отправит письмо-подтверждение на `info@example.org`, откройте его и подтвердите пересылку.

### Проверка и ограничения

- После настройки отправьте тестовое письмо на `source@example.com` и проверьте, что оно появилось в `info@example.org`.
- Не включайте удаление писем из `source@example.com` сразу. Сначала проверьте несколько тестовых писем.
- Письма из папки `Спам` обычно не пересылаются автоматически.
- Если правило не срабатывает, проверьте приоритет правил и нет ли выше правила, которое останавливает дальнейшую обработку.
